## 📋 專案概述

**專案名稱**：2026 Winter Europe Travel Itinerary Web  
**版本**：v1.0  
**建立日期**：2025-12-03  
**資料來源**：`z:\2026-Germany` 行程規劃檔案（Markdown 行程 & 景點說明）

---

## 🎯 專案目標

將 **2026 年德國南部冬季 15 天自駕行程**（1/26 - 2/9）轉換為互動式旅遊網頁，提供清晰、美觀、實用的行程資訊展示，方便旅行者在：

- 出發前快速規劃、檢查整體路線與安排
- 旅途中以手機查閱「今天行程」與景點、導航資訊
- 旅行後回顧並分享給旅伴或未來參考者

---

## 👥 使用者角色 (User Personas)

| 角色 | 描述 | 主要需求 |
|:---|:---|:---|
| **主要旅行者** | 行程規劃者本人 | 快速查閱每日行程、景點資訊、交通路線，旅途中以手機方便操作 |
| **旅伴** | 同行旅行者 | 了解行程安排、隨時打開連結就能看到今天要去哪、如何前往 |
| **未來參考者** | 計畫類似行程的旅行者 | 參考路線規劃、預算、實用提醒，快速理解整體行程架構 |

## 📖 User Stories

> 說明：以下維持原有 Epic / User Story 結構，在不改變編號的前提下，補充更清楚的驗收條件與技術細節。

### Epic 1：行程總覽

#### US-001：15 天行程時間軸
> **作為** 旅行者，  
> **我想要** 在首頁看到 15 天行程的視覺化時間軸，  
> **以便** 快速掌握整趟旅程的路線和重點。

**驗收條件 (Acceptance Criteria)**：
- [ ] 顯示 1/26 - 2/9 共 15 天的時間軸
- [ ] 每天顯示：日期、星期、主要城市、當日主題
- [ ] 點擊任一天可跳轉至該日詳細行程
- [ ] 支援響應式設計，手機上可垂直滾動瀏覽
- [ ] 標註特殊事件（跨境、火車日、長途駕駛日）

#### US-002：行程地圖總覽
> **作為** 旅行者，  
> **我想要** 在地圖上看到整趟行程的路線，  
> **以便** 了解地理位置和移動距離。

**驗收條件**：
- [ ] 顯示德國、奧地利、法國相關區域地圖
- [ ] 標註所有住宿城市（哈修塔特、符茲堡、海德堡、科爾馬、加米許、慕尼黑）
- [ ] 顯示自駕路線連線（以概略路線 / 折線表示，無須精準導航規劃）
- [ ] 地圖以 **Google Maps Embed** 互動地圖呈現，可縮放和平移
- [ ] 同時提供「在 Google Maps 開啟」按鈕，於手機上可直接開啟 App，作為嵌入地圖的備援

### Epic 2：每日詳細行程

#### US-003：每日行程頁面
> **作為** 旅行者，  
> **我想要** 查看每一天的詳細時間表，  
> **以便** 知道每個時段要做什麼。

**驗收條件**：
- [ ] 以時間表格式顯示（時間 | 活動 | 備註），時間欄可留空代表彈性時段
- [ ] 包含交通方式和預估車程時間
- [ ] 標註重要提醒（如：需提前預訂、冬季可能關閉）
- [ ] 顯示當日住宿資訊
- [ ] 可切換上一天/下一天
- [ ] 提供 **快速切換 Day 1 ~ Day 15** 的互動元件（例如：上方 Day 按鈕列或下拉選單），在桌機與手機上皆能方便選擇指定天數

#### US-004：景點資訊卡片
> **作為** 旅行者，  
> **我想要** 看到每個景點的詳細資訊卡片，  
> **以便** 了解開放時間、門票、特色等重要資訊，並能直接開啟導航。

**驗收條件**：
- [ ] 卡片包含：景點名稱、照片、簡介
- [ ] 顯示：開放時間、門票價格、地址
- [ ] 標註冬季特別注意事項
- [ ] 可展開/收合詳細資訊（手機上建議預設收合，減少頁面長度）
- [ ] 提供明顯的 **「開啟導航」按鈕**：
  - 桌機：點擊後以新分頁開啟對應的 Google Maps 網址
  - 手機：點擊後可開啟 Google Maps App 或瀏覽器導航頁面

## 🛠️ 技術需求

### 前端技術選擇

| 選項 | 技術棧 | 優點 | 缺點 |
|:---|:---|:---|:---|
| **方案 A** | 純 HTML/CSS/JavaScript | 實作與部署最簡單、無需建置流程、可直接 GitHub Pages 部署 | 需自行維護重複結構、複雜互動需手刻 |
| **方案 B** | React + Tailwind CSS | 元件化、互動性強、易於擴充 | 需要建置步驟與 Node.js 環境 |
| **方案 C** | Vue 3 + Vite | 輕量、快速開發 | 需要 Node.js 環境與建置流程 |

**建議選擇**：方案 A（純 HTML/CSS/JavaScript）
- 適合靜態旅遊資訊展示
- 可直接部署到 GitHub Pages
- 便於維護和分享

### 必要功能

- **響應式設計**：支援桌面、平板、手機，重點頁面（日行程、景點卡片）在手機直向使用時仍易於閱讀與操作
- **離線支援**：使用 Service Worker 快取首頁、每日行程頁面 HTML、CSS、JS，讓使用者在離線時仍可閱讀行程文字內容（地圖與部分圖片若無法載入需顯示提示訊息）
- **列印友善**：提供列印版面樣式（隱藏導覽列與不必要裝飾），支援 A4 直式列印
- **深色模式**：可選擇是否實作，若實作需確保文字對比度仍符合 WCAG 2.1 AA

### 外部資源

- **地圖**：
  - 行程總覽頁：使用 **Google Maps Embed API** 顯示互動地圖，可縮放和平移
  - 每日行程 / 景點卡片：透過 Google Maps 網址連結實作「開啟導航」按鈕（手機上可開啟 App）
  - 若未設定 API 金鑰或遇到流量限制，可退回為靜態地圖截圖 + 「在 Google Maps 開啟」按鈕作為備援
- **圖片**：景點照片需注意版權，建議優先使用 Unsplash/Pixabay 等免費圖庫或自有拍攝照片
- **圖示**：Font Awesome 或 Material Icons

### 內容產生流程（Markdown → 靜態網頁）

為了維持行程內容的可維護性，資料以 Markdown 原始檔為主，藉由簡單流程產出靜態 HTML：

1. 在 `z:\2026-Germany` 下維護行程與景點的 Markdown 檔（例如：`93-*.md`、各城市景點檔）。
2. 以簡單工具或腳本（可使用現成 Markdown 轉 HTML 工具，或透過 Gemini 協助）將 Markdown 轉為對應區塊的 HTML。
3. 將產生的 HTML 區塊套用至共用版型（含導覽列、頁尾、CSS），生成各頁靜態網頁檔案（例如 `index.html`、`day-01.html`、`day-02.html`）。
4. 將產出的靜態檔案放入 GitHub Pages 使用的目錄（例如 `/docs`），由 GitHub 自動發布。

> 目標是讓「修改 Markdown → 重新產出 HTML → 部署」的流程可在數步內完成，避免手動同步多份重複內容。

### Service Worker 更新策略

為避免離線快取導致使用者長期看到舊版本頁面，需定義簡單明確的更新策略：

- Service Worker 檔案內維護版本號常數（例如 `SW_VERSION`），每次發佈新版本時遞增。
- 新版 Service Worker 安裝完成後，使用 `skipWaiting` 與 `clientsClaim` 讓新版在使用者重新整理後即能生效。
- 初版可不做複雜 UI 提示；若未來需要，可在偵測到新版快取時，顯示「有新版本，請重新整理」的小提示列。

### （選配）多語系支援

- 若未來需要支援英文版，可採用前端字串表（JSON）方式，於導覽列右上角提供語言切換按鈕（例如：`繁中 / EN`）。
- 最低需求：切換時導覽列、主要標題、按鈕文字與關鍵說明文字可在中/英文之間切換；自由文字備註可視需求逐步補齊。
- 多語系屬於選配功能，建議待中文版穩定後再實作。

## 📅 開發里程碑與優先順序

| 階段 | 內容重點 | 目的 | 預估時間 |
|:---|:---|:---|:---|
| **Phase 1** | 專案架構、首頁、15 天時間軸、每日行程頁骨架（含 Day 快速切換）、GitHub Pages 初次部署 | 建立可用的 MVP 骨架，能瀏覽各天行程與基本資訊 | 2-3 天 |
| **Phase 2** | 完成 15 天每日行程內容與景點資訊卡片（含導航按鈕）、基本美術與排版調整 | 讓旅途中查閱「今天行程」已足夠實用 | 3-5 天 |
| **Phase 3** | 美食推薦、住宿資訊、交通路線總覽、預算頁面 | 提供完整的實用資訊與預算參考 | 2-3 天 |
| **Phase 4** | 行程地圖總覽整合、冬季自駕指南、細部 UX 優化 | 強化路線理解與冬季安全資訊 | 2-3 天 |
| **Phase 5** | 效能優化、離線支援（Service Worker）、列印樣式調整、多語系（若採用）、最終測試與部署 | 提升體驗品質與分享/離線能力 | 1-2 天 |

**MVP 範圍**（最小可用版本）：

- 首頁 + Hero 區 + 15 天時間軸
- 每日行程頁面（可先完成全部文字內容，景點卡片可使用簡化樣式）
- Day 快速切換元件
- 每個景點至少有一個可用的「開啟導航」按鈕
- GitHub Pages 部署與可分享網址
